# AlDente UX 패리티 점검

AlDente(AppHouseKitchen)의 메뉴바 UX와 volta의 `MenuBarExtra` 구성을 대조했습니다. (출처: AlDente 공식 features 페이지, GitHub 저장소)

## AlDente의 메뉴바 구성 (요지)

- **라이브 상태 아이콘**: 메뉴바 아이콘이 충전/연결 여부를 표시.
- **충전 상한 슬라이더**: 20~100%, 슬라이더 또는 숫자 입력. 팝오버.
- **Sailing Mode**: 하한을 둬서 micro-charging 방지(하한~상한 사이를 점선으로 표시).
- **Discharge 모드**: 연결 중에도 배터리만 사용해 강제 방전.
- **온도/상태 표시**, 과열 시 충전 일시정지(Heat Protection).
- **우클릭 빠른 토글**: 아이콘 우클릭으로 충전 일시정지/재개(상한을 현재%↔100% 토글).

## volta 현재 구성 대조

| 항목 | AlDente | volta 현재 | 판정/조치 |
|------|---------|-----------|-----------|
| 메뉴바 상태 아이콘 | 충전/연결 반영 | `menuBarSymbol`(상태별 심볼) | ✅ 반영(실기 검증) |

| Discharge | 토글 | 토글(목표 50% 고정) | △ 목표값 노출 권장 |

## 이번에 반영한 것

- 충전 상한 슬라이더 범위 **20~100%** (정책 Bounds·슬라이더·검증 테스트 동기화). 소프트웨어 충전-억제 방식이라 임의 상한 가능함을 UI 문구로 안내.

## 권장 개선(스코프 관리상 다음 단계로 — 새 위험 없이 UX만 향상)

1. **상한 숫자 입력 필드** 추가(슬라이더 옆 `TextField`). 정밀 입력.
2. **Sailing 하한 노출**: 현재 `chargeStartThreshold`(상한−5 자동)를 사용자 조절 가능한 두 번째 슬라이더/스테퍼로. 정책엔진은 이미 히스테리시스를 지원하므로 UI만 추가하면 됨.
3. **강제 방전 목표값 노출**: 현재 50% 고정 → 스테퍼로 조절(정책엔진은 임의 목표 지원).
4. **우클릭 빠른 일시정지/재개**: `MenuBarExtra`는 우클릭 커스터마이즈가 제한적이라, 메뉴 내 "지금 충전 일시정지/재개" 버튼으로 대체 제공 권장.
5. **메뉴바 텍스트에 % 직접 노출 옵션**(설정).

위 1~5는 기능 추가가 아니라 기존 정책엔진 능력의 UI 노출이라, 하드웨어 위험 없이 점진 적용 가능합니다.
