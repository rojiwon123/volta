# volta 기능 스펙

> macOS 배터리 충전 관리 메뉴바 앱. 이 문서는 사용자에게 제공하는 기능의 사양을 관리한다.

## 기기 지원 / Capability
**Apple Silicon 전용.** Intel·미등록 모델은 SMC 제어를 전부 비활성화(fail-safe). 판별은 아키텍처 + 모델 식별자 기준이며, 모델 allowlist + 검증 상태로 관리한다. 자세한 규칙·현재 기기 탐지값·검증 상태는 **[docs/device-support.md](device-support.md)** 참조.
- 현재 작업 기기: `Mac17,5` / arm64 / macOS 26.5.1 → supported(효과 미검증).
- 실기 SMC 효과 검증은 유료 팀 계정 도입 후 진행(현재 키 매핑만).

## 기능 동작 우선순위
높은 번호가 낮은 번호를 덮어쓴다(상위 우선).
1. 과열 보호
2. 강제 방전 / 외출 준비 (상호 배타 — 동시 활성 불가)
3. 수면 중 충전 중단
4. 충전 제한

참고: 2번(강제 방전/외출 준비)이 활성화되면 목표치 도달까지 수면이 자동으로 방지되므로(§5), 작동 중에는 시스템이 잠들지 않는다. 따라서 2번과 3번이 충돌하는 상황은 발생하지 않는다.

## 배터리 상태 표시 기준 (전역 원칙)
배터리 상태를 표시하는 모든 곳 — ① 맥 메뉴바 아이콘, ② 팝오버 상단 배터리 표시 — 는 전력 흐름 섹션(§9)과 완전히 동일한 기준으로 표시한다. 실제 전력 흐름을 근거로 충전중 / 유지 / 방전 중 하나로 표시하며, 어떤 기능이 켜져 있는지(설정)는 표시를 직접 좌우하지 않는다.
- 유일한 예외: 과열 보호 설정이 활성화되어 있고 임계 온도를 넘은 경우 → 해당 위치 모두 과열 보호 아이콘/표시로 바뀐다.
- 그 외 충전/유지/방전 상태는 항상 실제 전력 흐름 기준.

## 기능 목록
### 1. 맥 메뉴바 아이콘
배터리 상태를 직관적으로 확인할 수 있는 메뉴바 아이콘. 표시 기준은 위 "배터리 상태 표시 기준(전역 원칙)"을 따른다 — 전력 흐름 기반, 과열만 예외.
### 2. 팝오버
메뉴바 아이콘 클릭 시 열리는 패널. 다양한 설정과 세부 정보를 표시한다.
- 팝오버 상단 배터리 표시: 메뉴바 아이콘과 마찬가지로 위 "배터리 상태 표시 기준(전역 원칙)"을 따른다 — 전력 흐름(§9)과 완전히 동일한 기준으로 충전/유지/방전 표시, 과열 시에만 과열 표시.
### 3. 충전 제한
지정된 상한선까지만 충전하고 유지 상태로 전환한다.
- 설정: 상한선(max) 단일 값, 게이지 UI.
- 동작: 배터리 < max → max까지 충전 / 배터리 ≥ max → 유지(충전 억제, 능동 방전 없음).
- 참고: min(재충전 하한) 개념 없음 → 전력 재연결 시 max 미만이면 매번 max까지 충전된다.
### 4. 과열 보호
배터리가 지정된 온도를 넘어서면 발동(기능을 켜둔 경우에만).
- 설정: 임계 온도(30~60℃).
- 우선순위: 최상위. 발동 시 다른 기능을 모두 비활성화한다.
- 동작: 충전·강제 방전·외출 준비를 모두 중단하고 유지(idle) 상태로 식힌다. 식으면 정책 복귀.
- 근거: 배터리 발열은 전류가 내부 저항을 통과하며 생기는 줄열(I²R)이 주원인이다. 충전·방전 모두 전류가 흘러 열을 만들고, 특히 방전은 배터리가 시스템에 전류를 공급하므로 셀 발열을 더한다. 따라서 온도를 가장 빠르게 낮추는 상태는 전류가 최소인 유지(idle)다.
### 5. 강제 방전 / 외출 준비 (상호 배타)
둘은 동시에 활성화할 수 없다. UI는 3택 셀렉터(없음 / 강제 방전 / 외출 준비)로 하나만 선택한다.
- 외출 준비: 충전 제한 설정과 관계없이 100%까지 충전. 100% 도달해도 자동 종료하지 않고 사용자가 끌 때까지 유지.
- 강제 방전: 지정된 하한선까지 방전(게이지 UI). 하한 도달 시 1회성 완료로 자동 종료되고, 전력이 연결돼 있으면 충전 제한 정책으로 복귀해 max까지 충전.
- 수면 자동 방지(기본 동작): 강제 방전/외출 준비가 활성화되면 목표치(100% / 하한선) 도달까지 수면을 자동으로 방지한다. 사용자 토글이 아니라 기본 동작.
### 6. 수면 중 충전 중단
수면 상태에 들어갈 때 유지 상태로 만들어 과충전을 방지한다. 설정: on/off 토글.
### 7. 충전 중 수면 방지
충전이 충전 제한 상한선에 도달할 때까지 시스템이 수면에 들어가지 않도록 막는다.
### 8. 배터리 사이클 / 수명 % 표기
팝오버에 배터리 사이클 수와 수명%(설계 용량 대비 현재 최대 용량)를 표시.
### 9. 전력 흐름 UI
상세한 전력 흐름과 연결 상태를 가시적인 UI로 제공(어댑터→시스템/배터리 Sankey, W 단위, 어댑터 정격/공급 전력).
### 10. 점검(self-test)
사용자가 팝오버의 **"점검" 버튼**(헬퍼 상태 근처)으로 직접 실행해, 각 제어가 **이 기기에서 실제로 동작하는지** 확인한다. 제어 가능(헬퍼 연결+기기 지원)할 때만 활성.
- 동작: 제어 가능한 기능을 **순차로** — ① 충전 억제(CHTE) 적용 → 충전이 멈췄는지 관찰 → 복원, ② 강제 방전/어댑터 차단(CHIE→CH0J→CH0I) 적용 → 방전 방향인지 관찰 → 복원. 판정은 기존 행동 기반 효과검증(`ControlEffectVerifier`)의 정착+샘플 타이밍을 재사용한다.
- 결과: 단계별 **동작함 / 동작 안 함 / 판정 불가(사유)**. (ACLC 충전 LED는 시각 검증이라 자동 판정에서 제외 — "수동 확인" 안내.)
- 전제/안전: 두 단계 모두 **어댑터 연결 필요**(미충족 시 "판정 불가: 어댑터 필요"). 충전 억제는 적용 직전 충전 중이어야, 어댑터 차단은 방전 중이 아니어야 관찰 가능. 점검 중 **"점검 중…" 표시 + 컨트롤 잠금**, 끝/중단/오류 시 **항상 안전 상태로 복원**(충전 허용·어댑터 정상·억제 해제). 사용자 설정(상한/강제방전 목표 등)은 점검이 바꾸지 않는다. 점검 중에는 정상 폴링 적용을 멈춰 간섭을 막는다.
- 검증상태 연결: 충전 억제가 **동작 안 함**이면 `DeviceSupport`를 `.ineffective`로 강등(전체 제어 비활성), 모든 단계가 **동작함**이면 `mappedUnverified`→`verifiedOnHardware`로 승격. 강제 방전 단독 실패는 강등하지 않고 결과 보고로만 드러낸다. (런타임 상태 — 저장 안 함, 앱 재시작 시 base로 복귀.)
- 분리: 오케스트레이션/판정은 `VoltaCore.SelfTest`(순수 함수, 단위 테스트), 라이브 적용/관찰/복원만 앱(`BatteryMonitor.runSelfTest`). 실제 배터리를 잠깐 억제/방전하므로 **사용자 버튼으로만** 실행.
